⚠️ IMPORTANT: URL Decoding Issue

If your meeting folder URLs were saved with URL encoding (e.g., %20 for spaces, %5B for brackets), you need to decode them before using in SharePoint filters.

Symptom: Your SharePoint HTTP request returns 0 items even though folders exist.

Why: SharePoint's FileRef field stores decoded paths like Meetings/Budget Meeting- [Urgent], but your URL might be encoded like Meetings/Budget%20Meeting-%20%5BUrgent%5D.

Fix: Add an additional Compose action after Set_Folder_Path:

  1. After Set_Folder_Path, add a new Compose action
  2. Name it: Decode_Folder_Path
  3. Use Expression: uriComponentToString(outputs('Set_Folder_Path'))
  4. In Action 12 (Send_HTTP_to_SharePoint), use @{outputs('Decode_Folder_Path')} instead of @{outputs('Set_Folder_Path')}

Updated Uri for Action 12:

_api/web/lists/GetByTitle('Meetings')/items?$select=Id,FileRef,FileLeafRef,FSObjType,Modified,HasUniqueRoleAssignments&$filter=startswith(FileRef,'@{outputs('Decode_Folder_Path')}')&$top=500

7b ACTION 7b: Config_Site_Path_Prefix ⭐ NEW
Location: After Config_List_Title

🎯 Purpose:

Build the SharePoint site path prefix dynamically from the parsed folder URL. This is CRITICAL for the SharePoint filter to work correctly because SharePoint's FileRef field stores server-relative paths starting with /sites/...

📝 How to Add:

  1. Click + New step
  2. Search for: compose
  3. Select Compose

⚙️ Configuration:

Inputs

Use Expression Tab:

concat('/', outputs('Parse_Folder_URL')[3], '/', outputs('Parse_Folder_URL')[4])

What this does: Combines / + element[3] + / + element[4] to build the site path.

Example:
URL: https://ABCTest179.sharepoint.com/sites/Permission-Scanner-Test/Meetings/Budget Meeting
Array[3]: "sites"
Array[4]: "Permission-Scanner-Test"
Result: "/sites/Permission-Scanner-Test"

Different site automatically works:
URL: https://...sharepoint.com/sites/MyOtherSite/Documents/HR Files
Result: "/sites/MyOtherSite"

🔧 Rename:

Rename to: Config_Site_Path_Prefix

🚨 CRITICAL: Why This is Required

SharePoint stores file paths in the FileRef field as server-relative paths:

Example FileRef value: /sites/Permission-Scanner-Test/Meetings/Budget Meeting- [Urgent]/document.docx NOT just: Meetings/Budget Meeting- [Urgent]/document.docx

Your filter MUST include the /sites/... prefix or you'll get 0 results even when files exist!

Dynamic extraction ensures: The prefix automatically matches whatever site the folder URL points to.

✅ Verification: When you test the flow, this should show your site path (e.g., "/sites/Permission-Scanner-Test")

15 ACTION 15: Filter_By_Level (Depth Check)
Location: After Filter_Broken_Permissions

🎯 Purpose:

Filter items by folder depth level. The meeting record has a "Level" field that controls how deep the scan should go. This prevents scanning unnecessary subfolders.

📝 How to Add:

  1. Click + New step
  2. Search for: select
  3. Select Select (Data Operations)

⚙️ Configuration:

From

Use Expression Tab:

@body('Filter_Broken_Permissions')
Map

Switch to Edit in advanced mode and enter this complex expression:

@if(greaterOrEquals(sub(length(split(item()?['FileRef'], '/')), length(split(outputs('Set_Folder_Path'), '/'))), outputs('Get_Meeting_Record')?['body/crad9_level']), item(), null)

Expression Breakdown:

Step 1: split(item()?['FileRef'], '/') → Splits item path into array Example: "Meetings/Q1/Budget.docx" → ["Meetings", "Q1", "Budget.docx"] Step 2: length(...) → Counts elements (depth = 3) Step 3: split(outputs('Set_Folder_Path'), '/') → Splits base folder path Example: "Meetings/Q1" → ["Meetings", "Q1"] Step 4: length(...) → Counts elements (base depth = 2) Step 5: sub(itemDepth, baseFolderDepth) → Calculates relative depth (3 - 2 = 1) Step 6: outputs('Get_Meeting_Record')?['body/crad9_level'] → Gets configured scan depth from meeting record Example: 2 (scan 2 levels deep) Step 7: greaterOrEquals(relativeDepth, configuredLevel) → Checks if item is within allowed depth Example: 1 >= 2 → TRUE (include this item) Step 8: if(condition, item(), null) → Returns item if within depth, null if too deep

🔧 Rename:

Rename to: Filter_By_Level

⚠️ Important: This filter creates an array that may contain null values for items that are too deep. We'll filter these out in the next step.
