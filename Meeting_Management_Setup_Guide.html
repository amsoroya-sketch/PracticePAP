🔌 Step 2: Authorize Connections

Estimated Time: 10 minutes

📌 Why This is Important

Power Automate flows need authorized connections to:

  • Access Dataverse tables (Meeting, Agenda Item, etc.)
  • Create folders in SharePoint
  • Read SharePoint permissions

Without authorized connections, flows will show "Connector not found" errors.

Authorize All Connections

  1. Open the Solution
    • In Solutions list, click on CRM-Power BI-SharePoint Integration
    • The solution details page will open
  2. Navigate to Connection References
    • In the left navigation (Objects pane), click Connection References
    • You should see 4 connection references listed
    📸 Screenshot: Connection References view showing 4 items with "Off" status
  3. Authorize Dataverse Connection
    • Find the connection reference starting with: "Microsoft Dataverse SharePointPermissionScan..."
    • Click on it to open the edit panel
    • In the Connection dropdown, you'll see:
      • Option A: Existing connection (e.g., "DevTestUser@ABCTest179.onmicrosoft.com")
      • Option B: "+ New connection"
    • If existing connection available: Select it from dropdown
    • If no connection: Click "+ New connection":
      • A new window opens
      • Click Create
      • Sign in with your account
      • Return to previous window
      • Click the refresh icon next to Connection dropdown
      • Select the newly created connection
    • The panel will automatically save (Save button may be disabled - this is normal)
    • Close the panel
  4. Authorize SharePoint Connections (3 connections)
    • Repeat the process above for each SharePoint connection reference:
    • 1. "SharePoint CRMPowerBISharePointIntegration-..." (first one)
    • 2. "SharePoint CRMPowerBISharePointIntegration-..." (second one)
    • 3. "SharePoint SharePointPermissionScanner-e7162"
    • For each one:
      • Click the connection reference
      • Select existing SharePoint connection OR create new
      • If creating new:
        • Click "+ New connection"
        • Click Create
        • Sign in if prompted
        • Return and select the new connection
      • Close panel
  5. Verify All Connections Authorized
    • Back in Connection References list
    • Check that all 4 connection references show a connection (not "Off")
    • Status should change from "Off" to show connection details
    📸 Screenshot: All 4 connection references showing connected status

✅ Success Criteria

  • All 4 connection references show a connection (not "Off")
  • 1 Dataverse connection authorized
  • 3 SharePoint connections authorized (can all use the same SharePoint connection)
  • No error messages

⚠️ Troubleshooting Connections

  • Save button disabled: This is normal - connections save automatically
  • Connection dropdown empty: Click "+ New connection" to create one
  • New connection doesn't appear: Click the refresh icon next to dropdown
  • Sign-in popup blocked: Allow popups for make.powerapps.com

🎨 Step 3: Customize the Meeting Form

Estimated Time: 30-45 minutes

🎯 What You're Creating

A professional meeting form with 4 organized tabs:

  • Tab 1: Meeting Details - Basic meeting info and SharePoint integration
  • Tab 2: Agenda Items - Manage agenda with COI tracking
  • Tab 3: Meeting Members - Track attendees and RSVP status
  • Tab 4: Permission Scans - View permission audit history

Part A: Open the Form Editor

  1. Navigate to the Meeting Table
    • In the solution (CRM-Power BI-SharePoint Integration)
    • In Objects pane, click Tables
    • Find and click Meeting (logical name: crad9_meeting)
  2. Open Forms
    • In the Meeting table details, find the Forms section
    • You should see a form named "Information" or "Main"
    • Click on it
  3. Edit Form
    • Click Edit form (or "Edit form in new tab")
    • The form designer will open
    • You'll see the default form layout
    📸 Screenshot: Power Apps form designer with Meeting form open

Part B: Create Tab Structure

  1. Rename Existing Tab to "Meeting Details"
    • Click on the first tab (usually called "General" or similar)
    • In the properties panel on the right, find Label
    • Change to: Meeting Details
  2. Add "Agenda Items" Tab
    • Click + Component in the left panel
    • Scroll down and click 1-column tab
    • A new tab appears on the form
    • Click on the new tab
    • In properties panel, set Label: Agenda Items
  3. Add "Meeting Members" Tab
    • Click + Component again
    • Click 1-column tab
    • Click on the new tab
    • Set Label: Meeting Members
  4. Add "Permission Scans" Tab
    • Click + Component one more time
    • Click 1-column tab
    • Click on the new tab
    • Set Label: Permission Scans

✅ You should now have 4 tabs: Meeting Details, Agenda Items, Meeting Members, Permission Scans

Part C: Configure Meeting Details Tab

  1. Click on Meeting Details Tab
  2. Add Basic Information Section
    • If there's already a section, keep it and rename to "Basic Information"
    • If not, click + Component > 1-column section
  3. Add Fields to the Section
    • From the left panel, expand Table columns
    • Drag these fields onto the form (in this order):
      • crad9_newcolumn (Meeting Name)
      • crad9_meetingstartdate (Meeting Start Date)
      • crad9_meetingenddate (Meeting End Date)
      • crad9_level (Permission Level)
  4. Add SharePoint Integration Section
    • Below the first section, click + Component > 1-column section
    • Click the new section, set Label: SharePoint Integration
    • Drag these fields into this section:
      • crad9_isuserallowedspfolder (Create SharePoint Folder)
      • crad9_spfoldercreated (SP Folder Created)
      • crad9_folderurl (Folder URL)
      • crad9_lastscandate (Last Scan Date)
  5. Make Read-Only Fields
    • Click on crad9_spfoldercreated field
    • In properties, set Read-only: Yes
    • Repeat for:
      • crad9_folderurl
      • crad9_lastscandate

Part D: Add Agenda Items Subgrid

  1. Click on "Agenda Items" Tab
  2. Add Subgrid Component
    • Click + Component in left panel
    • Search for or scroll to find Subgrid
    • Click on Subgrid
    • A dialog appears: "Select subgrid data"
  3. Configure Subgrid
    • Table: Select Agenda Item (crad9_agendaitem)
    • Default view: Select Active Agenda Items
    • Show related records: Yes (automatically filters by parent meeting)
    • Click Done
  4. Customize Subgrid Properties
    • Click on the subgrid you just added
    • In properties panel:
      • Label: Agenda Items
      • Show label: Yes
      • Enable editing: Yes (allows inline editing)
      • Show quick create: Yes (allows adding items without leaving form)
  5. Edit View to Add COI Column
    • Click Edit view (in subgrid properties)
    • A view designer opens
    • Click + View column
    • Search for and add: Has Conflict of Interest (crad9_hasconflictofinterest)
    • Arrange columns in this order:
      • Sequence
      • Name
      • Description
      • Has Conflict of Interest ⬅️ NEW!
      • Folder Created
      • SP Folder URL
    • Click Save and close

Part E: Add Meeting Members Subgrid

  1. Click on "Meeting Members" Tab
  2. Add Subgrid
    • Click + Component > Subgrid
    • Table: Select Meeting Contact (crad9_meetingcontact)
    • Default view: Active Meeting Contacts
    • Show related records: Yes
    • Click Done
  3. Configure Subgrid
    • In properties:
      • Label: Meeting Members
      • Enable editing: Yes
      • Show quick create: Yes
  4. Customize View Columns
    • Click Edit view
    • Ensure these columns are visible:
      • Contact (lookup)
      • Role (Organizer/Attendee/Optional)
      • RSVP Status
      • SP Access Granted
    • Click Save and close

Part F: Add Permission Scans Subgrid

  1. Click on "Permission Scans" Tab
  2. Add Subgrid
    • Click + Component > Subgrid
    • Table: Select Scan (crad9_scan)
    • Default view: Active Scans
    • Show related records: Yes
    • Click Done
  3. Configure Subgrid
    • In properties:
      • Label: Permission Scan History
      • Enable editing: No (read-only)
      • Show quick create: No
  4. Customize View Columns
    • Click Edit view
    • Include these columns:
      • Name
      • Status
      • Scan Start Time
      • Items Scanned
      • Broken Permissions Found
      • Total Permissions Recorded
    • Click Save and close

Part G: Save and Publish the Form

  1. Save Your Work
    • Click Save button (top right)
    • Wait for "Saved" confirmation
  2. Publish the Form
    • Click Publish button (top right)
    • Wait for "Published successfully" message
  3. Close Form Designer
    • Click the Back arrow or close the tab

✅ Form Customization Complete!

  • Form has 4 tabs
  • Meeting Details tab shows basic info and SharePoint integration
  • Agenda Items subgrid includes "Has Conflict of Interest" column
  • Meeting Members subgrid shows contacts with roles
  • Permission Scans subgrid shows audit history
  • Form is saved and published

🔍 Troubleshooting Guide

Issue 1: Folder Not Created After Saving Meeting

Symptoms:

  • "SP Folder Created" stays unchecked after 2+ minutes
  • Folder URL remains empty

Diagnosis Steps:

  1. Verify Trigger Field
    • Open the meeting record
    • Confirm "Create SharePoint Folder" is ✅ checked
    • Confirm "SP Folder Created" is unchecked
  2. Check Flow Status
    • Go to Power Automate (make.powerautomate.com)
    • Click My flows
    • Find CreateMeetingSPFolder
    • Click on it
    • Click Run history
    • Check if there's a recent run for your meeting
  3. Review Flow Errors
    • If run exists and shows "Failed", click on it
    • Review error details
    • Common errors:
      • SharePoint connection unauthorized - Re-authorize connection
      • Folder already exists - Delete duplicate folder in SharePoint
      • Special characters in name - Meeting name contains /, \, :, or ?

Solutions:

  • Connection Issue: Go to Solutions > Connection References > Re-authorize SharePoint connection
  • Duplicate Folder: Delete folder from SharePoint, edit meeting to trigger flow again
  • Special Characters: Rename meeting to remove invalid characters (/, \, :, ?)
  • Flow Not Triggering: Ensure both conditions met: "Create SP Folder" = Yes AND "SP Folder Created" = No

Issue 2: Agenda Item Subfolder Not Created

Symptoms:

  • Agenda item "Folder Created" stays unchecked
  • SP Folder URL remains empty

Common Causes:

  • Parent meeting folder doesn't exist yet
  • Meeting doesn't have "Create SharePoint Folder" enabled
  • Agenda item created before meeting folder completed

Solutions:

  • Wait for Meeting Folder: Ensure parent meeting shows "SP Folder Created" = Yes before adding agenda items
  • Check Meeting Settings: Verify meeting has "Create SharePoint Folder" = Yes
  • Re-trigger Flow: Edit agenda item (change name slightly), save to trigger flow again
  • Manual Creation: If flow keeps failing, create subfolder manually in SharePoint and update the agenda item record

Issue 3: Permission Scanner Fails

Symptoms:

  • Flow run shows "Failed"
  • Scan record shows Status = Failed
  • Error message in scan record

Common Errors:

Error Message Cause Solution
"Folder URL is empty" Meeting doesn't have folder URL Ensure meeting folder created first
"401 Unauthorized" SharePoint connection not authorized Re-authorize SharePoint connection
"404 Not Found" Folder doesn't exist in SharePoint Verify folder URL is correct, folder exists
"Invalid Meeting ID" Wrong GUID provided Copy Meeting ID from URL or record

Issue 4: COI Field Not Showing on Agenda Item Form

Symptoms:

  • "Has Conflict of Interest" column missing from subgrid
  • Can't set COI when creating agenda items

Solutions:

  • Field exists but not on form:
    1. Open Agenda Item table
    2. Go to Forms section
    3. Edit the main form
    4. Drag "Has Conflict of Interest" field onto form
    5. Save and Publish
  • Not in subgrid view:
    1. Edit the Agenda Items subgrid on Meeting form
    2. Click "Edit view"
    3. Add "Has Conflict of Interest" column
    4. Save and close

Issue 5: "Connector Not Found" When Editing Flows

Symptoms:

  • Opening flow shows error
  • Actions show yellow warning triangles
  • Can't save or run flow

Solution:

This means connections aren't authorized. Follow Step 2 (Authorize Connections) again:

  1. Go to Solutions > CRM-Power BI-SharePoint Integration
  2. Click Connection References
  3. For each of 4 connections:
    • Click the connection reference
    • Select or create a connection
    • Close the panel
  4. Verify all 4 show connected status
